API Endpoints

Clean RESTful API for your Canvas integration

POST /auth

Authenticate with Canvas and receive your unique user ID

{
  "canvasUrl": "https://canvas.university.edu",
  "apiKey": "your_canvas_api_token",
  "institutionName": "University Name"
}
GET /courses/{userId}

Get all courses for authenticated user

GET /assignments/{userId}?course_id=123

Get assignments for a specific course

POST /canvas/{userId}

Proxy any Canvas API request

{
  "endpoint": "/courses",
  "method": "GET"
}

Get Started

Multiple deployment options for your workflow

💬 Claude Desktop (MCP)

Add to your Claude Desktop config. Restart Claude after saving. 

{
  "mcpServers": {
    "canvas-student": {
      "command": "npx",
      "args": ["-y", "@a-ariff/canvas-student-mcp-server"],
      "env": {
        "CANVAS_API_KEY": "your_canvas_api_token",
        "CANVAS_BASE_URL": "https://canvas.university.edu"
      }
    }
  }
}

📍 Get Canvas API token: Canvas → Account → Settings → Approved Integrations → New Access Token

🐳 Docker Deployment

Run with Docker for isolated deployment 

# Pull and run
docker run -d \
  -e CANVAS_API_KEY=your_token \
  -e CANVAS_BASE_URL=https://canvas.university.edu \
  -p 3000:3000 \
  canvas-mcp-server

☁️ Cloudflare Worker API

Use the hosted API endpoint 

# Authenticate
curl -X POST https://canvas-mcp.ariff.dev/auth \
  -H "Content-Type: application/json" \
  -d '{"canvasUrl": "https://canvas.university.edu", "apiKey": "your_token"}'

# Use your userId
curl https://canvas-mcp.ariff.dev/courses/YOUR_USER_ID
